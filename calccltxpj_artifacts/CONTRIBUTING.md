# Contribuindo

## Padrões
- PR pequeno e focado.
- Toda feature deve ter critérios de aceite e (quando possível) tracking.
- Nunca inserir segredos no repositório.

## Branch naming
- `feat/<slug>`
- `fix/<slug>`
- `chore/<slug>`

## Commits
Preferir Conventional Commits:
- `feat: ...`
- `fix: ...`
- `chore: ...`

## Segurança
- Chaves e tokens devem viver em variáveis de ambiente.
- IA (Gemini) deve ser chamada apenas via endpoint server-side.
